# RAG Assist Agent

RAG-бэкенд API (Retrieval-Augmented Generation), разработанный для бесшовной интеграции с **OpenWebUI**. Эта система использует **Gemini Embeddings**, векторную базу данных **Qdrant** и **LLM-реранжирование (OpenRouter / Qwen)** для предоставления точных ответов на основе ваших документов с учетом контекста. Система обладает продвинутой поддержкой парсинга графиков, таблиц и корректного извлечения и отображения изображений из PDF-файлов.

## Архитектура системы

1. **Парсинг документов (Docling + RapidOCR):** Генерация Markdown и JSON форматов с полным сохранением пространственной информации таблиц и извлечением изображений в высоком разрешении.
2. **Семантическое разбиение (Chunking):** Контекстно-зависимая нарезка документов на фрагменты до 512 токенов с отслеживанием связей "родитель-потомок".
3. **Векторизация (Embedding):** Использование модели `gemini-embedding-001` (через `google-genai`) с усечением размерности (MRL truncation).
4. **Векторный поиск (Retrieval):** Индексация и поиск по сходству через Qdrant (поддерживается как локальный In-Memory режим, так и Cloud DB).
5. **Реранжирование (Reranking):** Дополнительное переранжирование результатов с помощью быстрых LLM (через OpenRouter) для повышения релевантности.
6. **API слой:** Прокси-сервер на базе FastAPI, который имитирует эндпоинт `/v1/chat/completions` от OpenAI для 100% совместимости с OpenWebUI.

## Быстрый старт

### 1. Требования
- Python 3.10+
- `pip install -r requirements.txt`

### 2. Настройка окружения
Скопируйте шаблон `.env.example` и заполните ваши API-ключи:
```bash
cp .env.example .env
```
_Примечание: если вы используете OpenWebUI с мобильного телефона или внешнего IP-адреса, задайте переменную `BASE_IMAGE_URL` (например, `http://ВАШ_IP:8001`) в файле `.env`, чтобы картинки корректно загружались в браузере._

### 3. Запуск API Сервера
Запустите сервер Uvicorn FastAPI:
```bash
cd src
uvicorn api:app --host 0.0.0.0 --port 8001
```

### 4. Подключение к OpenWebUI
1. Перейдите в раздел **Настройки (Settings) > Подключения (Connections)** в интерфейсе OpenWebUI.
2. Добавьте новое соединение OpenAI с URL-адресом: `http://host.docker.internal:8001/v1` (или IP вашей машины, если сеть Docker не настроена, например `http://192.168.1.x:8001/v1`).
3. Введите любой фиктивный API ключ (например, `sk-12345`).
4. Выберите созданную модель `rag-assist-agent` в рабочей области чата и начните общение с вашими документами!
