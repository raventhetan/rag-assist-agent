# Инструкция по настройке интеграции RAG-пайплайна с OpenWebUI (Шаг 4)

Подробное руководство для настройки прокси-сервера RAG и подключения его в качестве модели к интерфейсу OpenWebUI.

## 1. Подготовка и запуск API прокси-сервера

Ваша задача — запустить скрипт `src/api.py`, который будет работать как сервер, совместимый с OpenAI API, перехватывая запросы OpenWebUI и пропуская их через RAG.

1. Убедитесь, что виртуальное окружение активировано (выполняется в папке проекта):
   ```bash
   source venv/bin/activate
   ```
2. Если вы еще не устанавливали зависимости, установите их:
   ```bash
   pip install fastapi uvicorn pydantic
   ```
3. Перейдите в папку `src`:
   ```bash
   cd src
   ```
4. Запустите сервер:
   ```bash
   uvicorn api:app --host 0.0.0.0 --port 8001
   ```
   *Опционально:* Рекомендуется настроить systemd сервис или использовать `tmux`/`screen` для поддержания работы сервера в фоне, чтобы он не отключался при закрытии терминала.

## 2. Настройка подключения в самом OpenWebUI

Теперь необходимо "познакомить" интерфейс OpenWebUI с нашим запущенным сервером. Войдите в веб-интерфейс OpenWebUI под учетной записью администратора.

1. Перейдите в **Settings** (Настройки) ⚙️ (в нижнем левом углу или контекстном меню вашего профиля).
2. Перейдите в раздел **Connections** (Подключения) или **API**.
3. Найдите секцию **OpenAI API**.
4. Нажмите иконку **+** (Добавить новое подключение).
5. В поле **API Base URL** (или Endpoint) введите адрес нашего сервера:
   ```text
   http://localhost:8001/v1
   ```
   *(Если OpenWebUI установлен в Docker-контейнере, а прокси-сервер крутится прямо на хостовой машине Linux, возможно, вместо `localhost` нужно будет использовать IP-адрес хост-машины в докер-сети, например `http://172.17.0.1:8001/v1` или `host.docker.internal`).*
6. В поле **API Key** введите любой случайный набор символов (например, `sk-12345`). Наш скрипт не проверяет ключи, но интерфейс OpenWebUI может требовать, чтобы поле не было пустым.
7. Нажмите **Save** (Сохранить) для проверки подключения. Значок рядом с подключением должен загореться зеленым или белым, что означает успешный пинг.

## 3. Интеграция изображений (Локальный файловый сервер)

> [!WARNING]
> **Известная проблема:** В настоящее время (03.03.2026) API-прокси успешно формирует markdown-ссылки на изображения на локальном сервере, однако картинки по-прежнему не отображаются в интерфейсе OpenWebUI. Данная проблема находится в стадии решения (возможно связано с CORS или парсером OpenWebUI).

В данный момент `api.py` возвращает локальные файловые пути к изображениям. Чтобы картинки-схемы действительно отображались в браузере внутри OpenWebUI, их нужно отдавать по HTTP.

Для этого:
1. Запустите простой статический HTTP сервер для папки `output/images`:
   ```bash
   python -m http.server 8080 -d ../output/images
   ```
2. В файле `src/api.py` (если это будет необходимо вашему проекту) пути картинок нужно будет формировать как веб-ссылки:
   ```markdown
   ![Схема](http://localhost:8080/image_name.png)
   ```

## 4. Создание и настройка модели (Модели в OpenWebUI)

1. Перейдите в список доступных моделей в интерфейсе чата (Workspace -> Models, либо просто выпадающий список сверху).
2. Наш API отдает модель с названием `volhovec-rag` (имя задано внутри `api.py`).
3. Обязательно выберите эту модель в качестве активной для текущего чата.
4. Напишите тестовый запрос, например: *"Какие размеры у двустворчатых дверей типа 3?"* и убедитесь, что:
   * Ответ формируется и опирается на базу знаний.
   * Стриминг (посимвольное появление текста) работает плавно, как у ChatGPT.
   * В консоли сервера `uvicorn` появляются подробные логи от Qdrant и Gemini.
